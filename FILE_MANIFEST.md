## How Files Work Together

### Discovery Flow
```
tap_perplexity/__init__.py
  └─> tap_perplexity/streams.py (load schemas)
      └─> tap_perplexity/schemas/*.json
```

### Sync Flow
```
tap_perplexity/__init__.py
  └─> tap_perplexity/client.py (API calls)
  └─> tap_perplexity/streams.py (stream logic)
      └─> tap_perplexity/schemas/*.json
```

### Testing Flow
```
tests/unit/test_*.py
  └─> tap_perplexity/*.py (test targets)
      
tests/integration/test_*.py
  └─> tap_perplexity/*.py (test full flow)
  └─> config.json (API credentials)
```

## Maintenance

### Adding New Files

When adding new files, update:
1. This manifest
2. MANIFEST.in (if needed for distribution)
3. .gitignore (if should be excluded)
4. setup.py (if new package)

### File Naming Conventions

- **Documentation**: UPPERCASE.md
- **Python modules**: lowercase_with_underscores.py
- **Test files**: test_*.py
- **Schemas**: lowercase.json
- **Scripts**: lowercase.sh
